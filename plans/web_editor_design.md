# Web 编辑器详细设计

> **国际化约定**：编辑器代码中所有用户可见文本（按钮、标签、提示、菜单项等）应集中管理（如统一的字符串常量对象），便于后期实现多语言本地化。当前版本语言统一使用**中文**。
> **节点命名约定**：节点内部类型 ID 可继续使用英文（如 `CheckHP`），但所有面向用户显示的节点名称都应走本地化显示名，不直接显示内部英文类型名。
> **搜索约定**：节点搜索必须同时支持“本地化显示名”和“内部英文类型名/英文别名”。例如中文环境下 `CheckHP` 显示为“检测HP”时，搜索“检测”和搜索“check”都应命中该节点。

> **渲染架构**：采用 HTML 节点 + Canvas 连线方案。节点为 HTML div 元素（原生支持表单控件、文本选择、输入法），Canvas 仅用于绘制背景网格和连线。单文件实现，无外部依赖。

# 第一部分：设计文档

## 1. 整体美术风格

**风格定位**：Blueprint 蓝图风格 — 专业节点编辑器感，参考 UE5 Blueprint / Blender Shader Editor。

**色彩体系**：
- 画布背景：深灰蓝 `#1a1a2e`，细网格线 `#222240`，粗网格线 `#2a2a45`
- 面板背景：`#12121e`，面板边框 `#2a2a45`
- 文字主色：`#d4d4e8`，次要文字：`#6c6c8a`，禁用/提示：`#44445a`
- 强调色（交互反馈）：`#5a9bf6`（蓝）
- 危险/删除：`#e05555`
- 类别色：
  - Composite: `#3a7ebf`（蓝）
  - Decorator: `#4a9a6a`（绿）
  - Action: `#c07838`（橙）
  - Condition: `#b89a30`（黄）

**字体**：`'Segoe UI', system-ui, sans-serif`，节点标题 12px bold，节点摘要/属性 11px regular，面板标题 13px，面板内容 12px。

**圆角**：节点 6px，面板 0px（贴边），按钮 4px，输入框 3px，弹窗 8px。

**阴影**：节点投射 `2px 3px 8px rgba(0,0,0,0.4)`，悬停时加深。选中节点边框发光（类别色 + box-shadow glow）。

**表单控件风格**：保持 Blueprint 风格的克制深色外观。输入框/下拉框背景使用深色纯色或轻微层次，不使用浏览器原生高亮外观；边框默认低对比，hover 略提亮，focus 使用强调色细描边。

## 3. 中央画布（Canvas）

### 3.1 背景网格
- 两层网格：小格 40px（细线 `#222240`）+ 大格 200px（粗线 `#2a2a45`）
- 随缩放和平移同步移动
- 原点位置可画十字标记（可选）

### 3.2 节点

**结构（折叠态）**：
```
      ● ← 输入端口（顶部居中，悬停区域 20px，可见圆点 8px）
╭────────────────────╮
│ ■   Sequence     ▶ │ ← 标题栏：类别色填充，左侧小方块图标（可切换激活/关闭），
│                    │   标题文字居中，右侧三角图标（可展开节点才有）
├────────────────────┤
│ 3 children         │ ← 摘要区：深色半透明背景，文字色 #a0a0be
╰────────────────────╯
      ● ← 输出端口（底部居中，仅 composite/decorator）
```

- Root 节点：左侧小方块替换为金色 `★`，Root 节点不可关闭
- 非 Root 节点：左侧小方块可点击切换激活/关闭（后续实现）
- 节点数据增加 `enabled` 字段，表示该节点是否启用，默认值为 `true`
- 当节点关闭（`enabled: false`）时，运行时跳过该节点及其整个子树，效果等同于这棵子树不存在
- 若一个已关闭节点被设为 Root，则该节点自动恢复为开启状态
- 单选 Root 节点时不可修改启用状态；多选时启用/关闭开关始终可编辑，但若选区中包含 Root 节点，则该修改对 Root 节点不生效

**结构（展开态）**：
```
      ●
╭────────────────────╮
│ ■ InjectInput      │ ← 标题栏
├────────────────────┤
│ target:  [P2    ▾] │ ← 展开区：显示全部属性
│ inputs:  [FORWARD] │    可直接在节点内编辑
│ frames:  [60     ] │
│ relative:[true  ▾] │
╰────────────────────╯
      (无输出端口, action 节点)
```

**折叠/展开规则**：
- 双击节点标题栏切换折叠/展开
- 单击标题栏右侧的三角图标（`▶` / `▼`）切换折叠/展开
- 无属性或属性极少的节点（Sequence, Selector, Parallel, Noop）不支持展开，始终显示标题+摘要，不显示三角图标
- 可展开节点在折叠态标题栏右侧显示 `▶`，展开态显示 `▼`

**节点宽度**：固定 160px。高度动态：标题栏 28px + 摘要行 18px（折叠）或每属性行 24px（展开）。

**摘要文案**（临时方案，各节点具体显示内容待节点设计文档中定义）：
| 节点类型 | 折叠摘要 |
|---------|---------|
| Sequence / Selector / Parallel | `N children` |
| RandomSelector | `N children (random)` |
| Repeat | `×N` 或 `forever` |
| WaitFrames | `30f` |
| InjectInput | `P2: FWD+LP 60f` |
| Noop | (无摘要，仅标题) |

> **注**：每个节点的摘要格式应在节点设计文档中详细定义，编辑器实现时从节点定义中读取摘要规则。

**视觉状态**：
- 默认：类别色标题 + 深色内容体 + 细边框 `#2a2a45`
- 悬停：边框变亮 `#505068`，阴影显著加深
- 选中：类别色发光边框（2px + glow shadow），标题栏微亮
- Root 节点：标题栏左侧用 `★` 替代小方块图标，Root 节点不可关闭
- 小方块图标：可点击激活/关闭节点（后续实现），Root 节点无小方块
- 关闭态：节点整体变灰；连线仍然显示，用于保留树结构的可读性

### 3.3 端口（Ports）

- 输入端口：节点顶部居中，可见圆点 ⌀8px（填充 `#3a3a55`，边框 `#555`），悬停热区 ⌀20px
- 输出端口：节点底部居中，同上（仅 composite/decorator 有）
- 悬停：圆点放大至 ⌀10px，填充变为类别色，外圈发光
- 连线模式中：所有合法目标端口脉冲高亮（类别色 + 呼吸效果）

### 3.4 连线（Connections）

- 贝塞尔曲线，从父节点输出端口到子节点输入端口
- 默认颜色：`#3a3a55`，线宽 2px
- 悬停：变为父节点的类别色，线宽 3px；若父节点有多个子节点，同时高亮该父节点的所有子连线（兄弟连线亮度低于悬停连线，约 50% 透明度）
- 悬停连线时按 Delete 键 → 断开该连接
- 流动方向：仅悬停时显示——沿曲线移动的小圆点（⌀4px），类别色 50% 透明度，约 2 秒一个完整周期
- 多个子节点时，连线旁标注序号（1, 2, 3...），小字 `#6c6c8a`
- 正在拖拽创建连线时：虚线 + 源节点类别色高亮，线宽 3px；
- 连线拖拽命中节点时，不要求必须落在端口上；拖到节点本体也可自动连到该节点对应的输入/输出端口

### 3.5 画布交互

| 操作 | 行为 |
|------|------|
| 左键拖拽节点 | 移动节点（释放时吸附网格） |

## 5. 右侧面板（Properties Editor）

### 5.1 布局（从上到下）
- **顶部**：折叠按钮（`»` / `«`）
- **树属性区**：树名称、描述、onComplete 选择
- **分隔线**
- **节点属性区**：选中节点的详细信息
  - 只读信息（与可编辑属性在样式上明确区分，无输入框/控件，纯文本显示）：
    - Type：主文字色 `#d4d4e8`，字号 13px bold，最醒目
    - Category：次要文字色 `#6c6c8a`，字号 12px，类别色小圆点标记
    - ID：最淡文字色 `#44445a`，字号 11px，低优先级信息
  - 启用/关闭开关：
    - 单选非 Root 节点时可编辑
    - 单选 Root 节点时禁用或隐藏该控件
    - 多选时始终可批量编辑；若包含 Root 节点，则 Root 节点不受此次修改影响
  - Root 标记 / 设为 Root 按钮
  - 子节点列表（支持拖拽排序 + ↑↓按钮排序，可断开 ×）
    - 每条目高度 28px，左侧拖拽手柄 `⠿`，类别色竖条 + 节点名称，右侧 ×
    - 列表宽度撑满面板（留 8px 左右内边距），确保可读性和操作空间
  - 可编辑属性（input/select/checkbox 控件）
    - 数字输入框隐藏浏览器原生上下箭头
    - 下拉框使用自定义深色弹出菜单，不使用浏览器原生菜单样式
    - 下拉菜单高亮分三级：其他选项无高亮、已选项为较暗高亮、鼠标悬停项为更亮高亮
  - 删除节点按钮
- **无选中时**：灰色提示文字 "Click a node to edit"
- **多选同类型节点时**：显示该类型的属性编辑界面
  - 若某属性在所有选中节点中值相同，显示该值
  - 若某属性在选中节点中值不同，显示 "多个值"
  - 修改属性时，值同时应用到所有选中节点
- `enabled` 为所有节点共有属性，不受“必须同类型”限制；多选时始终允许批量编辑
- **多选不同类型节点时**：显示 "已选择 N 个节点（多种类型）" + 批量删除按钮

### 5.2 折叠与调宽
- 折叠时宽度收为 0，画布自动扩展
- 左边缘可拖拽调整宽度

### 5.3 与画布展开态的关系
- 节点在画布上展开编辑属性时，右侧面板同步显示同一节点的属性
- 两处修改实时同步
- 右侧面板始终显示完整属性（包括子节点列表等画布节点内不显示的内容）

## 7. 系统功能（全局性，不属于单一区域）

### 7.1 Undo/Redo
- 全状态快照方式，最多 50 条
- 所有变更操作前 push undoStack
- 拖拽仅在 mouseup 时记录
- Ctrl+Z 撤销，Ctrl+Shift+Z 重做

### 7.2 LocalStorage 自动保存
- 每次变更后 debounce 500ms 保存
- 页面加载时检测并弹出恢复提示条（顶部横幅："Unsaved tree recovered" + Load / Discard）
- 保存内容：树数据 + 视口状态（panX/panY/zoom）
- `beforeunload` 提示防止意外关闭

### 7.3 吸附网格
- 释放节点时自动吸附到 40px 网格
- 按住 Shift 拖拽禁用吸附
- 工具栏 Snap 按钮全局开关

### 7.4 框选（Marquee Select）
- 左键在空白处拖拽画选择框（蓝色半透明矩形）
- 多选后可批量移动、删除
- Shift+点击追加/移除选中
- 属性面板显示 "N nodes selected" + 批量删除按钮

### 7.5 复制/粘贴
- Ctrl+C 序列化选中节点（仅选中节点，不含子树）
- Ctrl+V 生成新 ID，偏移 +20,+20 放置
- 支持多选复制

### 7.6 验证
- 实时检测：不可达节点、空 Decorator、循环引用、缺少必填属性
- 有问题的节点右上角显示红/黄三角图标
- 工具栏可加 Validate 按钮列出所有问题

### 7.7 JSON 格式
- 导出格式与 `bt_schema.lua` 完全兼容
- 节点位置存在 `display: {x, y}` 字段中
- 折叠/展开状态存在 `display.collapsed` 中
- 节点启用状态导出为 `enabled: true/false`
- 即使节点关闭也必须导出，运行时按“跳过该节点及其整个子树”处理
- 运行时忽略所有 `display` 字段

# 第二部分：实现步骤

> 以下步骤按依赖顺序排列。每步完成后应能独立测试验证，不依赖后续步骤。

## Step 1：基础骨架与布局系统

**目标**：搭建三栏布局 + Canvas 层 + 主题样式，页面可正常显示。

**内容**：
1. HTML 结构：左面板、中央画布区（Canvas + 节点容器 div）、右面板
2. CSS：全局样式变量（色彩体系、字体、圆角等），flex 三栏布局
3. 左右面板折叠按钮（`«`/`»`）功能
4. 拖拽分隔条调整面板宽度（min/max 限制）
5. Canvas 铺满画布区，绘制双层背景网格
6. 国际化字符串常量对象（`I18N = { ... }`），所有用户可见文本从中读取

**验证**：页面加载后三栏正确显示，面板可折叠/展开，分隔条可拖拽调宽，网格随窗口大小自适应。

## Step 2：画布平移与缩放

**目标**：画布视口可自由移动和缩放。

**内容**：
1. 视口状态：`panX`, `panY`, `zoom`
2. 中键拖拽平移
3. 右键拖拽平移（移动 >3px 判定为拖拽，不触发右键菜单）
4. 滚轮缩放（朝鼠标位置，范围 0.2x ~ 3x）
5. 网格随平移/缩放同步更新
6. 节点容器 div 通过 CSS transform 同步视口变换

**验证**：中键/右键拖拽平移流畅，滚轮缩放以鼠标为中心，网格跟随。

## Step 3：节点数据模型与渲染

**目标**：能在画布上创建和显示 HTML 节点。

**内容**：
1. 节点数据结构：`{ id, type, category, enabled, properties, children, display: {x, y, collapsed} }`
2. 节点类型注册表（类型名 → 类别、默认属性、是否可展开、摘要生成函数）
   - 同时包含本地化显示名，以及用于搜索的英文别名/内部类型名
3. 注册当前所有节点类型（Sequence, Selector, Parallel, RandomSelector, Repeat, WaitFrames, InjectInput, Noop 等）
4. 节点 DOM 渲染：
   - 标题栏（类别色背景 + 图标 + 名称 + 三角图标）
   - 摘要区（折叠态）
   - 属性编辑区（展开态，input/select/checkbox 控件）
   - 输入端口（顶部居中）、输出端口（底部居中，仅 composite/decorator）
5. 节点视觉状态 CSS：默认、悬停、选中、Root 标记（`★`）、关闭态（整体变灰）
6. 节点定位：absolute 定位于节点容器 div 内，坐标为画布坐标

**验证**：手动调用创建节点函数，节点正确显示在画布上，随视口变换移动。

## Step 4：节点交互（选中、移动、折叠、启用开关）

**目标**：节点可选中、拖拽移动、折叠/展开，并可切换启用/关闭状态。

**内容**：
1. 左键点击节点 → 选中（高亮边框 + 右面板联动占位）
2. 左键拖拽节点 → 移动（实时更新位置）
3. 双击标题栏 / 单击三角图标 → 切换折叠/展开
4. 节点属性在展开态直接可编辑（修改后更新数据模型 + 摘要刷新）
5. 点击空白区域 → 取消选中
6. 点击标题栏左侧小方块 → 切换节点启用/关闭；Root 节点不可关闭，被设为 Root 时自动恢复启用

**验证**：节点可选中高亮、拖拽移动流畅、折叠/展开切换正常、展开态属性可编辑，小方块开关行为符合 Root 与 enabled 规则。

## Step 5：连线系统

**目标**：节点间可通过端口创建贝塞尔连线。

**内容**：
1. Canvas 绘制层：专门绘制连线（在网格之上、节点之下或之上均可，建议在节点 div 下方）
2. 贝塞尔曲线绘制：从父节点输出端口到子节点输入端口
3. 连线默认样式：`#3a3a55`，2px
4. 拖拽创建连线：从输出端口开始拖拽 → 虚线 + 源节点类别色 → 松开到合法输入端口 → 建立连接
5. 端口悬停反馈：放大 + 类别色填充 + 发光
6. 连线模式下合法目标端口脉冲高亮
7. 多子节点时连线旁标注序号
8. 数据模型更新：父节点 `children[]` 中添加/移除子节点 ID
9. 节点位置变化时重绘连线（监听节点位置）

**验证**：可拖拽创建连线，连线正确跟随节点移动，端口有悬停反馈。

---

## Step 6：连线悬停与删除

**目标**：连线可悬停高亮、删除。

**内容**：
1. 连线碰撞检测：沿贝塞尔曲线采样 20+ 点，鼠标距曲线 <5px 判定为悬停
2. 悬停高亮：父节点类别色 + 线宽 3px
3. 兄弟连线联动高亮（50% 透明度弱高亮）
4. 悬停时显示流动动画（沿曲线移动圆点，2 秒周期）
5. 悬停连线时按 Delete → 断开连接
6. 连线上右键 → Delete Connection 菜单

**验证**：悬停连线时正确高亮（含兄弟连线），流动动画显示，Delete 和右键可删除。

## Step 7：Undo/Redo

**目标**：所有操作可撤销/重做。

**内容**：
1. 全状态快照栈（undoStack / redoStack，最多 50 条）
2. 所有变更操作前 push 快照：添加/删除节点、移动节点（mouseup 时）、修改属性、创建/删除连线、设置 Root
3. Ctrl+Z 撤销、Ctrl+Shift+Z 重做
4. 工具栏 Undo/Redo 按钮（栈空时 disabled）
5. 从快照恢复时重建所有节点 DOM 和连线

**验证**：各操作均可撤销/重做，连续撤销/重做状态正确。

## Step 13：复制/粘贴/剪切

**目标**：Ctrl+C/V/X 节点操作。

**内容**：
1. Ctrl+C：序列化选中节点（仅选中节点，不含子树），存入内部剪贴板
2. Ctrl+V：反序列化，生成新 ID，默认放置在当前鼠标位置
3. Ctrl+X：复制后删除原节点
4. 支持多选复制/剪切
5. 所有操作推入 undo 栈
6. Undo 规则补充：
   - 影响画布/treeData 的复制粘贴相关操作（如 Duplicate、Paste、Paste Properties、Cut 中的删除部分）进入 undo 栈
   - 仅修改内部剪贴板内容的操作（如 Copy / Copy to Clipboard 本身）不进入 undo 栈
   - Undo/Redo 不回滚内部剪贴板内容，只回滚画布与树数据状态

**验证**：复制/粘贴/剪切功能正确，ID 不重复，undo 可恢复。
